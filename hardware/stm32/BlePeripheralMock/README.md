# SmartBLE STM32 Hardware Reference

本目录包含了 SmartBLE 官方提供的下位机 C 语言硬件参考实现（基于意法半导体 STM32 架构）。本工程不仅为前端（Flutter/UniApp/Desktop）提供了标准的测试硬件锚点，同时**完全严格遵循**团队的《开发环境安装说明（团队通知版）》工程规范。

## 一、架构设计理念

为了满足在企业级开发中“底层配置剥离”与“业务逻辑内聚”的要求，本工程采用了现代化的 **STM32CubeMX + CMake + VS Code** 分层架构：

### 目录规范：
- `.ioc`：唯一硬件映射源（引脚重映射、时钟树配置、中间件勾选全在 CubeMX 中固化）。禁止手工编写对应的底层 HAL 初始化代码。
- `Core/`：由 CubeMX 自动生成的内核文件，**开发者禁止在此目录做大幅度修改**。
- `App/`：业务流核心。包含自定义的无阻塞状态机，设备配网逻辑等。
- `Bsp/`：单板外设驱动包（LED, 传感器与看门狗）。禁止向上依赖 `App` 层，必须保证高内聚低耦合。
- `Protocols/`：**核心区**。跨平台全端复用的通讯协议体，如 `smart_ble_protocol` 定义了前端发送 OTA 包的格式、指令与 UUID 表。

## 二、首次联板验证清单 (Bring-up Checklist)

新加入团队的成员或试图进行前后端联调测试的人员，在 Clone 代码库后，**必须在当日完成以下基础验证**：

- [ ] **A. 工具链通过**：VS Code 能正常识别 `.vscode/launch.json`。
- [ ] **B. 编译通过**：执行 **CMake Configure** 和 **Build All** 任务，控制台提示 `build finished successfully`。
- [ ] **C. 联机固化**：ST-LINK 接上目标板，执行 Debug 任务，程序能停在 `main()` 函数体首行。
- [ ] **D. 前端回环测试**：程序全速运行后，按复位键。打开电脑/手机端 Smart BLE 应用，能扫描到名为 `SmartBLE-STM32` 的外围设备，并在连接后直接向 `BEB5483E-...` 发送测控协议。

## 三、依赖扩展与芯片配置

本架构目前的预设环境为**双芯片透传方案**：
- **主控芯片**：`STM32F103C8T6` (CMake 工具链中宏定义为 `STM32F103xB`，Cortex-M3)。
- **蓝牙外设**：基于标准的透传模块（例如 `JDY-23` 或 `HC-08`）。

在这种双芯片结构下，您的 STM32 不需要移植任何庞大臃肿的软协议栈。您只需要：
1. 取一个 USART 串口连接蓝牙模块。
2. 用 `HAL_UART_Transmit` 替代原生蓝牙底层推流接口。
3. 监控模块的 `STAT` 状态引脚高低电平来判断手机是否已连接。

所有的核心业务数据分发仍将受 `App/` 和 `Protocols/smart_ble_protocol` 控制，这就完成了 STM32F103 与跨端应用软件的闭环。

## 四、标准接线方案 (Wiring Guide)

为保证在您的 STM32CubeMX 工程中能直接复用本文档逻辑，请让外设/硬件工程师按照以下经典跳线标准组装：

### JDY-23（或 HC-08）蓝牙模块 与 STM32F103 最小系统板

| STM32F103 引脚 (功能)      | 线序走向 | 蓝牙模块 (JDY-23) 引脚 | 作用描述说明 |
| :---                       | :---:   | :---                   | :---         |
| **3V3** (必须使用3.3V)         |  ───>   | **VCC**                | 为蓝牙模块供电，严禁接入 5V 否则可能烧毁 |
| **GND**                    |  ───>   | **GND**                | 共地连接 |
| **PA9**  (`USART1_TX`)     |  ───>   | **RXD**                | 单片机发送数据 -> 蓝牙模块接收后发给手机 |
| **PA10** (`USART1_RX`)     |  <───   | **TXD**                | 手机发来数据 -> 蓝牙模块发送给单片机接收 |
| **PA0**  (`GPIO_Input`)    |  <───   | **STAT**               | **连接状态标志**：高电平代表手机已连上，低电平未连接 |
| **PB12** (`GPIO_Output`)   |  ───>   | **EN (可选)**          | **休眠控制**：如果无需低功耗控制，此引脚悬空即可 |

**💡 避坑提示：**
1. 串口永远是 **交叉接线** (TX 接 RX，RX 接 TX)。
2. 建议大家不要在淘宝贪便宜买无名模块，买标准打有 `JDY-23` 或带有金属屏蔽罩的 `E104-BT50`，可以少走极大弯路，直接上电即用！
3. 请在 `STM32CubeMX` 中为 `PA9/PA10` 开启全局中断 (`NVIC: USART1 global interrupt`)，这是保证能实时收取 `SmartBLE Protocol` 并解包的关键。
