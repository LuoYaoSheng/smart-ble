# ESP32 BLE 服务器

基于 ESP32 的蓝牙低功耗服务器，支持 LED 控制和状态监控。

## 功能特点

- 支持 BLE 服务和特征值管理
- 支持 LED 控制（开/关/快闪/慢闪）
- 支持设备状态监控
- 支持 JSON 格式数据交互
- 支持多服务多特征值
- 支持实验性 BLE OTA 升级服务

## 硬件要求

- ESP32 开发板
- LED（连接到 GPIO2）

## 开发环境

- 开发框架：Arduino
- 开发工具：PlatformIO
- 分区表：`partitions_ota.csv`（双 OTA 分区）
- 依赖库：
  - NimBLE
  - ArduinoJson

## 服务说明

### 服务1：灯控服务 (UUID: 914b)
- 特征值 A8：控制特征值（常亮模式）
  - 权限：读写+通知
  - 功能：控制 LED 常亮/关闭
  - 命令：
    - 开灯/FF01：LED 常亮
    - 关灯/FF00：LED 关闭
- 特征值 A9：通知特征值（慢闪模式）
  - 权限：读写+通知
  - 功能：控制 LED 慢闪/关闭
  - 命令：
    - 开灯/FF01：LED 慢闪
    - 关灯/FF00：LED 关闭

### 服务2：权限演示服务 (UUID: 914c)
- 特征值 B0：只读特征值
- 特征值 B1：只写特征值
- 特征值 B2：只通知特征值
- 特征值 B3：读写特征值
- 特征值 B4：读和通知特征值
- 特征值 B5：写和通知特征值
- 特征值 B6：读写和通知特征值

### 服务3：OTA 服务
- 控制特征值：接收 `start` / `commit` / `abort` JSON 指令
- 数据特征值：接收固件分包
- 状态特征值：通知 `ready` / `progress` / `success` / `error`

## 数据格式

### 写入数据格式
- 文本格式：直接输入文本，如"开灯"、"关灯"
- HEX格式：输入十六进制数据，如"FF01"、"FF00"

### 响应数据格式
```json
{
    "type": "write_response",
    "command": "FF01",
    "led_state": "on",
    "blink_pattern": 1,
    "mode": "常亮"
}
```

## 编译和上传

1. 安装 PlatformIO
2. 打开项目
3. 编译项目
4. 上传到 ESP32

## 调试说明

1. 串口监视器波特率：115200
2. 启动指示：LED 慢闪 3 次
3. 连接状态：通过通知特征值发送
4. 设备状态：每 5 秒发送一次

## 注意事项

1. 确保正确连接 LED
2. 注意供电稳定性
3. 建议使用外部供电
4. 保持固件更新
5. OTA 仍为教学版实现，建议先在测试板上验证
