# Smart BLE 功能规格文档 (FRD)

> 版本：v1.0
> 更新日期：2024-12-30
> 项目：Smart BLE - 跨平台蓝牙调试工具

---

## 1. 项目概述

### 1.1 项目定位
Smart BLE（原 BLE Toolkit+）是一款专业的低功耗蓝牙（BLE）调试工具，旨在帮助开发者进行蓝牙设备的开发、测试和调试。项目未来将支持多平台实现，包括 uni-app、Flutter、原生 Android/iOS 和桌面端。

### 1.2 目标用户
- **主要用户**：蓝牙设备开发工程师
- **次要用户**：物联网开发者、嵌入式工程师
- **学习用户**：BLE 初学者、学生

### 1.3 支持平台
| 平台 | 当前状态 | 说明 |
|------|----------|------|
| 微信小程序 | ✅ 支持 | 完整功能支持 |
| Android App | ✅ 支持 | 需要定位和蓝牙权限 |
| iOS App | ✅ 支持 | 需要蓝牙权限 |
| H5 网页 | ⚠️ 有限 | Web Bluetooth API |
| Flutter | 🚧 规划中 | 支持移动端和桌面端 |
| 原生 Android | 🚧 规划中 | Kotlin 实现 |
| 原生 iOS | 🚧 规划中 | Swift + SwiftUI |
| 桌面端 | 🚧 规划中 | Flutter Desktop |

## 3. 详细功能说明

### 3.1 设备扫描功能

#### 3.1.1 功能描述
扫描附近的 BLE 设备，显示设备列表，支持多种过滤条件。

#### 3.1.2 功能特性
- **自动扫描**：点击"搜索设备"按钮开始扫描
- **自动停止**：默认 5 秒后自动停止扫描
- **节流处理**：1 秒间隔更新设备列表，防止 UI 卡顿
- **设备去重**：同一设备只显示最新信息

#### 3.1.3 过滤选项
| 过滤类型 | 范围/选项 | 说明 |
|----------|-----------|------|
| 信号强度过滤 | -100dBm ~ 0dBm | 只显示不低于设定值的设备 |
| 名称前缀过滤 | 任意字符串 | 只显示名称以该前缀开头的设备 |
| 隐藏无名称设备 | 开关 | 过滤掉没有本地名称的设备 |

#### 3.1.4 设备信息展示
每条设备记录包含：
- **设备名称**：本地名称（无名称显示"未知设备"）
- **设备类型标签**：根据名称自动识别（BLE/BT）
- **设备ID**：显示后 12 位
- **信号强度**：dBm 值 + 信号格数（1-4 格）

#### 3.1.5 信号格数计算
```
RSSI >= -60dBm  → 4格（强信号）
RSSI >= -70dBm  → 3格（良好）
RSSI >= -80dBm  → 2格（一般）
RSSI <  -80dBm  → 1格（弱信号）
```

#### 3.1.6 设备数量限制
- 最多显示 100 个设备（按信号强度排序）
- 超出数量后只保留信号最强的设备

#### 3.1.7 广播数据详情
点击设备卡片可查看广播数据详情：
- 设备 ID（完整）
- 设备名称
- RSSI 值
- 广播服务 UUID 列表
- 广播数据（Hex 格式）

---

### 3.2 设备连接功能

#### 3.2.1 功能描述
连接到选定的 BLE 设备，进入设备详情页进行服务发现和特征值操作。

#### 3.2.2 连接流程
```
用户点击"连接" → 跳转设备详情页 → 自动开始连接
                                   ↓
                            发起 BLE 连接请求
                                   ↓
                              连接成功/失败处理
                                   ↓
                          发现服务和特征值
```

#### 3.2.3 连接状态
- **未连接**：显示"连接"按钮
- **连接中**：显示加载状态
- **已连接**：显示"已连接"按钮（禁用状态）
- **连接失败**：显示错误日志

#### 3.2.4 自动重连
- 连接失败时不自动重连（用户手动重试）
- 连接断开时不自动重连

---

### 3.3 服务与特征值功能

#### 3.3.1 服务发现
连接成功后自动发现设备所有服务，每个服务包含：
- **服务 UUID**：完整 UUID 格式
- **特征值列表**：该服务下的所有特征值

#### 3.3.2 标准服务识别
| UUID | 服务名称 |
|------|----------|
| 1800 | 通用访问 (Generic Access) |
| 1801 | 通用属性 (Generic Attribute) |
| 180A | 设备信息 (Device Information) |
| 180F | 电池服务 (Battery Service) |
| 1812 | 人机接口设备 (HID) |
| 1819 | 位置和导航 (Location & Navigation) |

#### 3.3.3 特征值操作
每个特征值根据其属性支持以下操作：

##### 读取 (Read)
- 操作：点击"读取"按钮
- 结果：在日志中显示读取到的数据
- 数据格式：UTF-8 字符串或 Hex

##### 写入 (Write)
- 操作：点击"写入"按钮 → 弹出输入框 → 输入数据
- 支持格式：
  - **UTF-8 文本**：直接输入字符串
  - **Hex 数据**：输入 16 进制字符串（如 "FF01"）
- 结果：在日志中显示写入成功/失败

##### 通知/监听 (Notify)
- 操作：点击"监听"/"停止监听"按钮
- 状态：监听中显示"停止监听"，未监听显示"监听"
- 数据接收：实时在日志中显示接收到的数据

#### 3.3.4 特征值属性
| 属性 | 说明 | 操作按钮 |
|------|------|----------|
| Read | 可读 | 读取 |
| Write | 可写 | 写入 |
| Notify | 可通知 | 监听 |
| Indicate | 可指示 | 监听 |

### 3.4 BLE 广播功能

#### 3.4.1 功能描述
将设备作为 BLE 外设（Peripheral），对外广播自身信息，供其他设备扫描和连接。

> **注意**：此功能需要原生插件支持（LysBlePeripheral）

#### 3.4.2 支持平台
- ✅ 微信小程序（原生 API）
- ✅ Android App（原生插件）
- ✅ iOS App（原生插件）
- ❌ H5 网页（不支持）

#### 3.4.3 广播参数配置

##### 通用参数
| 参数 | 说明 | 示例 |
|------|------|------|
| 设备名称 | 广播时显示的设备名 | BLEToolkit_Android |
| 服务 UUID | 广播包含的服务 UUID | FFE0 |
| 厂商 ID | 16 位厂商 ID（Hex） | 0001 |
| 厂商数据 | 自定义厂商数据 | BLEToolkit_Test |

##### Android 专用参数
| 参数 | 选项 | 说明 |
|------|------|------|
| 广播模式 | 低功耗/平衡/低延迟 | 控制广播频率和功耗 |
| 发射功率 | 超低/低/中/高 | 控制广播信号强度 |
| 可连接 | 是/否 | 是否允许其他设备连接 |
| 包含设备名称 | 是/否 | 广播数据中是否包含设备名 |
| 添加服务 UUID | 是/否 | 广播数据中是否包含服务 UUID |

##### iOS 专用参数
| 参数 | 说明 |
|------|------|
| 本地名称 | 设备本地名称 |
| 服务列表 | 要广播的服务 UUID 数组 |
| 厂商数据 | 厂商 ID 和数据 |

#### 3.4.4 广播数据大小限制
- **总限制**：31 字节
- **设备名称**：建议不超过 8 字节
- **厂商数据**：建议不超过 4 字节
- 超出限制时自动截断数据

#### 3.4.5 广播状态
- **未广播**：显示"开始广播"按钮
- **广播中**：显示"停止广播"按钮
- **状态检查**：可点击"检查状态"查看当前广播状态

---

### 3.5 操作日志功能

#### 3.5.1 功能描述
记录所有 BLE 操作的日志，便于调试和问题排查。

#### 3.5.2 日志分类
| 类型 | 颜色 | 说明 |
|------|------|------|
| 系统 | 蓝色 | 系统级操作日志 |
| 错误 | 红色 | 错误信息 |
| 读取 | 绿色 | 数据读取操作 |
| 写入 | 橙色 | 数据写入操作 |
| 接收 | 紫色 | 接收到的通知数据 |

#### 3.5.3 日志格式
```
[时间] [类型] 消息内容
```
示例：
```
[14:23:15] 系统  开始搜索设备
[14:23:16] 读取  开始读取特征值：0000FFE1
[14:23:17] 接收  收到数据：Hello
```

#### 3.5.4 日志管理
- **自动滚动**：新日志自动滚动到底部
- **清除日志**：点击"清除"按钮清空所有日志
- **日志限制**：最多保留 100 条日志（新日志挤掉旧日志）

## 5. 数据格式

### 5.1 UUID 格式
- **完整格式**：`0000xxxx-0000-1000-8000-00805F9B34FB`
- **短格式**：`xxxx`（标准 UUID 的后 4 位）
- **自定义格式**：128 位随机 UUID

### 5.2 数据编码格式

#### UTF-8 文本
- 字符串直接转换为 ArrayBuffer
- 每个字符占 1-3 字节（取决于字符）

#### HEX 格式
- 输入：`"FF01"`（16 进制字符串）
- 转换：每 2 个字符转换为 1 字节
- 示例：`"FF01"` → `[0xFF, 0x01]`

### 5.3 ESP32 硬件协议

#### LED 控制命令
| 命令 | HEX | 效果 |
|------|-----|------|
| 开灯（常亮） | FF 01 | LED 常亮 |
| 关灯 | FF 00 | LED 关闭 |
| 快闪 | FF 02 | LED 200ms 闪烁 |
| 慢闪 | FF 03 | LED 1000ms 闪烁 |

#### JSON 响应格式
```json
{
    "type": "write_response",
    "command": "FF01",
    "led_state": "on",
    "blink_pattern": 1,
    "mode": "常亮"
}
```

## 7. 错误处理

### 7.1 常见错误码

#### 微信小程序
| 错误码 | 说明 | 处理 |
|--------|------|------|
| 10001 | 蓝牙未开启 | 提示用户开启蓝牙 |
| 10002 | 无蓝牙权限 | 引导用户授权 |
| 10004 | 已达到最大连接数 | 提示断开其他设备 |

#### Android
| 错误 | 说明 | 处理 |
|------|------|------|
| 缺少定位权限 | 蓝牙扫描需要定位权限 | 引导用户授权 |

#### iOS
| 错误 | 说明 | 处理 |
|------|------|------|
| 未授权 | 蓝牙权限被拒绝 | 引导用户到设置 |

### 7.2 错误提示原则
- **明确**：清楚告知用户问题
- **可操作**：提供解决方案或操作按钮
- **友好**：避免技术术语，使用用户易懂的语言

## 8. 性能要求

### 8.1 扫描性能
- 扫描延迟：< 1 秒

### 8.2 连接性能
- 连接超时：10 秒
- 服务发现超时：10 秒
- 读写操作超时：5 秒

### 8.3 内存要求
- 设备列表缓存：最多 100 个设备
- 日志限制：最多 100 条
- 广播数据缓存：实时更新
