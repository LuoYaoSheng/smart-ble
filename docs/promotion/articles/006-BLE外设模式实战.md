# BLE 外设模式实战：让你的手机变身蓝牙设备，互相发现！

## 前言

你知道你的手机可以变身蓝牙设备吗？

利用 BLE 外设模式（Peripheral Mode），**手机 A 可以广播，手机 B 可以扫描发现它**！

这在很多场景都很有用：
- 测试蓝牙扫描功能
- 模拟物联网设备
- 设备间通信
- 室内定位

---

## 一、什么是外设模式？

### 1.1 BLE 两种角色

```
┌─────────────┐                    ┌─────────────┐
│             │                    │             │
│   Central   │     扫描/连接      │  Peripheral │
│   (中心设备)  │ <──────────────> │   (外设)    │
│             │                    │             │
└─────────────┘                    └─────────────┘
   手机/电脑                          手机/ESP32
```

| 角色 | 说明 | 例子 |
|------|------|------|
| **Central** | 扫描、连接设备 | 调试 App、电脑 |
| **Peripheral** | 广播、被连接 | 手环、传感器 |

### 1.2 Smart BLE 支持外设模式

| 平台 | 外设模式支持 | 说明 |
|------|-------------|------|
| uni-app | ✅ | 微信小程序/App |
| Flutter | ✅ | Android/iOS/macOS |
| Electron | ✅ | Linux 需要 bleno |
| Tauri | ❌ | btleplug 限制 |
| macOS Native | ✅ | 原生支持 |

## 二、实战：两台手机互相发现

### 场景

手机 A：开启外设模式，广播 "HelloDevice"
手机 B：扫描，发现 "HelloDevice"

## 三、手机 A：开启广播

### Step 1: 打开外设模式

在 Smart BLE App 中：

1. 进入"广播"页面
2. 点击"开始广播"

### Step 2: 配置广播参数

```
📡 广播配置
├── 设备名称: HelloDevice
├── 服务 UUID: 4FAFC201-1FB5-459E-8FCC-C5C9C331914B
├── 厂商数据: FF FF AB CD
└── 可连接: 是
```

**参数说明**：

| 参数 | 说明 | 示例 |
|------|------|------|
| 设备名称 | 其他设备看到的名称 | MyDevice |
| 服务 UUID | 你提供的服务 | 自定义 128 位 UUID |
| 厂商数据 | 自定义数据 | 最大 31 字节 |
| 可连接 | 是否允许被连接 | 通常选是 |

### Step 3: 开始广播

点击"开始广播"，手机 A 开始发送信号：

```
📡 正在广播...
├── 名称: HelloDevice
├── 状态: 广播中
└── 已广播 23 秒
```

## 四、手机 B：扫描发现

### Step 1: 开始扫描

在 Smart BLE App 中：

1. 进入首页
2. 点击"开始扫描"

### Step 2: 查找设备

在设备列表中找到 "HelloDevice"：

```
📱 扫描结果

└── Sensor_1234         -72 dBm

### Step 3: 连接设备

点击 "HelloDevice"，手机 B 连接到手机 A

### Step 4: 发现服务

手机 B 会看到手机 A 广播的服务：

```
🔍 服务列表
└── Custom Service
    ├── Read Characteristic
    ├── Write Characteristic
    └── Notify Characteristic
```

## 六、进阶：模拟 IoT 设备

### 场景

模拟一个温度传感器，每 5 秒广播一次温度

### ESP32 模拟器代码（Flutter）

```dart
class TemperatureSensorSimulator {
  final BLEPeripheral peripheral;
  Timer? _timer;

  void startAdvertising() async {
    // 配置广播
    await peripheral.startAdvertising(
     AdvertisingOptions(
        name: 'TempSensor_01',
        serviceUUIDs: ['00001809-0000-1000-8000-00805f9b34fb'], // 温度服务
      ),
    );

    // 开始模拟温度变化
    _timer = Timer.periodic(Duration(seconds: 5), (timer) {
      _sendTemperature();
    });
  }

  void _sendTemperature() {
    // 生成随机温度 20-30°C
    final temp = 20 + Random().nextDouble() * 10;

    // 按照蓝牙标准格式编码
    final data = _encodeTemperature(temp);

    // 通知订阅者
    peripheral.notify(data);
  }

  List<int> _encodeTemperature(double temp) {
    // 蓝牙温度格式：IEEE-11073 32-bit float
    final bytes = ByteData(4);
    bytes.setFloat32(0, temp, endian: Endian.little);
    return bytes.buffer.asUint8List().toList();
  }
}
```

### 扫描端读取温度

```
📡 温度服务 (0x1809)
└── 温度测量特征值
    ├── 读取
    └── 订阅通知

订阅后收到：
🌡️ 温度: 25.3°C
🌡️ 温度: 24.8°C
🌡️ 温度: 26.1°C
```

## 八、注意事项

### 权限问题

| 平台 | 需要的权限 |
|------|-----------|
| Android | BLUETOOTH_ADVERTISE |
| iOS | NSBluetoothAlwaysUsageDescription |
| 微信小程序 | 外设模式权限 |

### 兼容性

| 平台 | 限制 |
|------|------|
| 微信小程序 | 需要用户授权，部分功能受限 |
| Android | 某些厂商系统限制后台广播 |
| iOS | 后台广播有时间限制 |

### 性能建议

```
✅ 推荐做法
- 广播间隔适中（100-500ms）
- 数据包小于 31 字节
- 不需要时停止广播

❌ 避免做法
- 广播间隔过短（< 50ms）
- 发送大量数据
- 长时间持续广播
```

## 九、完整示例

### 模拟一个智能灯泡

```dart
class SmartBulbSimulator {
  bool isOn = false;
  int brightness = 100;
  int color = 0xFFFFFF; // RGB

  void start() async {
    // 开始广播
    await peripheral.startAdvertising(AdvertisingOptions(
      name: 'SmartBulb_RGB',
      serviceUUIDs: ['4FAFC201-1FB5-459E-8FCC-C5C9C331914B'],
    ));

    // 监听写入
    peripheral.onWriteReceived((data) {
      _handleCommand(data);
    });
  }

  void _handleCommand(List<int> data) {
    final cmd = data[0];

    switch (cmd) {
      case 0x01: // 开关
        isOn = data[1] == 0x01;
        break;
      case 0x02: // 亮度
        brightness = data[1];
        break;
      case 0x03: // 颜色
        color = (data[1] << 16) | (data[2] << 8) | data[3];
        break;
    }

    // 通知状态变化
    _notifyState();
  }

  void _notifyState() {
    final state = [
      0x01, // 状态报告
      isOn ? 0x01 : 0x00,
      brightness,
      (color >> 16) & 0xFF,
      (color >> 8) & 0xFF,
      color & 0xFF,
    ];

    peripheral.notify(state);
  }
}
```
