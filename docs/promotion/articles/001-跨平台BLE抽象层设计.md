# 如何设计一个跨平台 BLE 抽象层？Smart BLE 架构设计揭秘

## 前言

在开发 Smart BLE 的过程中，最大的挑战之一就是：**如何让 8+ 种平台共享一套统一的 BLE 代码逻辑？**

uni-app 用微信小程序 API，Flutter 用 flutter_blue_plus，桌面端用 noble/btleplug...每个平台的 API 都不一样，怎么办？

答案就是：**设计一个跨平台 BLE 抽象层**。

---

## 一、为什么需要抽象层？

### 1.1 痛点

不同平台的 BLE API 差异巨大：

```typescript
// 微信小程序
wx.startBluetoothDevicesDiscovery({
  success: (res) => { }
})

// flutter_blue_plus
await flutterBluePlus.startScan(timeout: Duration(seconds: 4));

// noble (Node.js)
noble.startScanningAsync([], false);
```

如果没有抽象层，业务逻辑代码会变得混乱：

```typescript
// 糟糕的代码 - 到处都是平台判断
if (platform === 'wechat') {
  wx.startBluetoothDevicesDiscovery()
} else if (platform === 'flutter') {
  await flutterBluePlus.startScan()
} else if (platform === 'electron') {
  noble.startScanning()
}
// ... 到处都是这种判断
```

### 1.2 抽象层的好处

- ✅ 业务逻辑统一，只需写一次
- ✅ 新增平台只需实现接口
- ✅ 代码更易维护和测试
- ✅ 降低学习成本

## 二、抽象层设计

### 2.1 核心接口定义

```typescript
/**
 * BLE 适配器统一接口
 * 所有平台实现都必须遵循这个接口
 */
interface IBLEAdapter {
  // ========== 初始化 ==========
  initialize(): Promise<void>
  getState(): Promise<AdapterState>

  // ========== 扫描 ==========
  startScan(options?: ScanOptions): Promise<void>
  stopScan(): Promise<void>
  onDeviceFound(callback: (device: ScanResult) => void): void

  // ========== 连接 ==========
  connect(deviceId: string): Promise<void>
  disconnect(deviceId: string): Promise<void>
  onConnectionStateChanged(callback: (state: ConnectionState) => void): void

  // ========== 服务发现 ==========
  discoverServices(deviceId: string): Promise<BLEService[]>

  // ========== 读写操作 ==========
  readCharacteristic(serviceId: string, charId: string): Promise<DataBuffer>
  writeCharacteristic(
    serviceId: string,
    charId: string,
    data: DataBuffer,
    writeType?: WriteType
  ): Promise<void>

  // ========== 通知 ==========
  setNotification(
    serviceId: string,
    charId: string,
    enable: boolean
  ): Promise<void>
  onCharacteristicChanged(callback: (data: CharacteristicData) => void): void

  // ========== 广播（外设模式）==========
  startAdvertising(options: AdvertisingOptions): Promise<void>
  stopAdvertising(): Promise<void>

  // ========== 清理 ==========
  cleanup(): Promise<void>
}
```

### 2.2 类型定义

```typescript
// 适配器状态
enum AdapterState {

## 三、平台实现

### 3.1 uni-app 实现

```typescript
class UniAppBLEAdapter implements IBLEAdapter {
  async initialize(): Promise<void> {
    return new Promise((resolve, reject) => {
      wx.openBluetoothAdapter({
        success: () => resolve(),
        fail: (err) => reject(err)
      })
    })
  }

  async startScan(options?: ScanOptions): Promise<void> {
    return new Promise((resolve, reject) => {
      wx.startBluetoothDevicesDiscovery({
        services: options?.serviceUUIDs,
        allowDuplicatesKey: options?.allowDuplicates ?? false,
        success: () => {
          // 监听扫描结果
          wx.onBluetoothDeviceFound((res) => {
            res.devices.forEach(device => {
              this.onDeviceFoundCallback?.(this.parseDevice(device))
            })
          })
          resolve()
        },
        fail: (err) => reject(err)
      })
    })
  }

  async connect(deviceId: string): Promise<void> {
    return new Promise((resolve, reject) => {
      wx.createBLEConnection({
        deviceId,
        success: () => resolve(),
        fail: (err) => reject(err)
      })
    })
  }

  // ... 其他方法实现
}
```

### 3.2 Flutter 实现

```dart
class FlutterBLEAdapter implements IBLEAdapter {
  final FlutterBluePlus _flutterBlue = FlutterBluePlus.instance;

  @override
  Future<void> initialize() async {
    // 检查蓝牙状态
    await _flutterBlue.isOn;
  }

  @override
  Future<void> startScan([ScanOptions? options]) async {
    await _flutterBlue.startScan(
      timeout: Duration(seconds: options?.timeout ?? 5),
      withServices: options?.serviceUUIDs
        ?.map((uuid) => Guid(uuid))
        .toList(),
    );

    // 监听扫描结果
    _flutterBlue.scanResults.listen((results) {
      for (ScanResult r in results) {
        onDeviceFoundCallback?.call(_parseDevice(r));
      }
    });
  }

  @override
  Future<void> connect(String deviceId) async {
    await _flutterBlue.connect(deviceId);
  }

  // ... 其他方法实现
}
```

### 3.3 Electron (Noble) 实现

```typescript
class NobleBLEAdapter implements IBLEAdapter {
  private noble = require('@abandonware/noble');

  async initialize(): Promise<void> {
    return new Promise((resolve, reject) => {
      this.noble.on('stateChange', (state) => {
        if (state === 'poweredOn') resolve()
      })
    })
  }

  async startScan(options?: ScanOptions): Promise<void> {
    this.noble.on('discover', (peripheral) => {
      this.onDeviceFoundCallback?.(this.parseDevice(peripheral))
    })
    this.noble.startScanning(
      options?.serviceUUIDs ?? [],
      options?.allowDuplicates ?? false
    )
  }

  async connect(deviceId: string): Promise<void> {
    const peripheral = await this.findPeripheral(deviceId)
    return new Promise((resolve, reject) => {
      peripheral.connect((error) => {
        error ? reject(error) : resolve()
      })
    })
  }

## 四、使用抽象层

### 4.1 业务逻辑统一

有了抽象层，业务逻辑只需要写一次：

```typescript
class BLEManager {
  private adapter: IBLEAdapter

  constructor(adapter: IBLEAdapter) {
    this.adapter = adapter
  }

  // 扫描并连接第一个匹配的设备
  async scanAndConnect(serviceUUID: string): Promise<void> {
    // 1. 初始化
    await this.adapter.initialize()

    // 2. 扫描
    await this.adapter.startScan({ serviceUUIDs: [serviceUUID] })

    // 3. 等待发现设备
    return new Promise((resolve) => {
      this.adapter.onDeviceFound((device) => {
        if (device.serviceUUIDs.includes(serviceUUID)) {
          // 4. 连接
          this.adapter.connect(device.id).then(() => {
            this.adapter.stopScan()
            resolve()
          })
        }
      })
    })
  }

  // 读取特征值
  async readCharacteristic(serviceId: string, charId: string) {
    return await this.adapter.readCharacteristic(serviceId, charId)
  }

  // 写入特征值
  async writeCharacteristic(
    serviceId: string,
    charId: string,
    data: DataBuffer
  ) {
    await this.adapter.writeCharacteristic(serviceId, charId, data)
  }
}
```

### 4.2 切换平台

切换平台只需要更换适配器：

## 五、设计原则

### 5.1 接口最小化

只定义核心方法，保持接口简洁：

```typescript
// ✅ 好的设计 - 核心方法
interface IBLEAdapter {
  connect(deviceId: string): Promise<void>
  disconnect(deviceId: string): Promise<void>
}

// ❌ 不好的设计 - 过于具体
interface IBLEAdapter {
  connectWithTimeout(deviceId: string, timeout: number): Promise<void>
  connectWithRetry(deviceId: string, retry: number): Promise<void>
  connectAndDiscover(deviceId: string): Promise<void>
}
```

### 5.2 异步优先

所有操作都返回 Promise，适应现代异步编程：

```typescript
// ✅ 好的设计
async connect(deviceId: string): Promise<void>

// ❌ 不好的设计
connect(deviceId: string, callback: (error: Error) => void): void
```

### 5.3 错误统一

统一的错误处理机制：

```typescript
class BLEError extends Error {
  constructor(
    public code: BLEErrorCode,

## 六、总结

一个好的跨平台抽象层应该：

1. **接口简洁** - 只定义核心方法
2. **类型安全** - 完整的类型定义
3. **异步优先** - 适应现代编程模式
4. **易于扩展** - 新增平台成本低
5. **错误友好** - 统一的错误处理
