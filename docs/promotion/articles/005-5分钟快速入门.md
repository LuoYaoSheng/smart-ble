# 5 分钟上手：用 Smart BLE 调试你的第一个蓝牙设备

## 前言

刚拿到一个蓝牙设备，不知道怎么调试？

今天教你用 Smart BLE 在 **5 分钟内**完成第一次蓝牙调试！

---

## 一、准备工作

### 1.1 你需要

| 项目 | 说明 |
|------|------|
| 智能手机 | Android 或 iOS |
| 蓝牙设备 | 任何 BLE 设备（手环、灯泡、传感器等）|
| Smart BLE | 微信小程序扫码即可 |

### 1.2 微信扫码

打开微信扫描下方二维码，进入 Smart BLE 小程序：

```
[小程序二维码]
```

或者直接搜索 "Smart BLE"

---

## 二、第一次扫描（1 分钟）

### Step 1: 开启蓝牙

1. 打开 Smart BLE 小程序
2. 首次进入会提示开启蓝牙
3. 点击"允许"授权

### Step 2: 开始扫描

1. 点击首页的"开始扫描"按钮
2. 等待设备列表出现

### Step 3: 查看结果

你会看到附近所有 BLE 设备：

```
📱 设备列表
├── MyBand_AB34          -65 dBm
├── SmartLight_1234      -72 dBm
├── Unknown_Device       -85 dBm
└── ESP32_Demo           -55 dBm  ← 信号最强
```

**小技巧**：
- 信号越强，距离越近（dBm 越大越好）
- 点击右上角可过滤设备名称

---

## 三、连接设备（1 分钟）

### Step 1: 选择设备

点击你想连接的设备，例如 "ESP32_Demo"

### Step 2: 等待连接

设备旁边显示"连接中..."，大约 1-3 秒后会变成"已连接"

### Step 3: 查看服务

连接成功后，自动跳转到服务列表：

```
🔍 服务列表
├── Generic Access (0x1800)
├── Device Information (0x180A)
└── Custom Service (4FAF...)
    ├── Control Characteristic
    ├── Data Characteristic
    └── Status Characteristic
```

**小知识**：
- **0x1800** 是标准服务号
- 以 4FAF 开头的是自定义服务

---

## 四、读取设备信息（1 分钟）

### Step 1: 找到设备信息服务

展开 "Device Information" 服务

### Step 2: 读取特征值

找到特征值，点击"读取"按钮

### Step 3: 查看结果

你会看到设备返回的信息：

```
📋 读取结果
├── 制造商: Espressif
├── 型号: ESP32-C3
├── 固件版本: 1.0.0
└── 序列号: ABC123456
```

---

## 五、写入数据（1 分钟）

### Step 1: 找到可写特征值

在自定义服务中，找到带有"Write"属性的特征值

### Step 2: 选择数据格式

Smart BLE 支持两种格式：

- **UTF-8**：文本格式，如 `on`、`off`
- **HEX**：十六进制，如 `FF 01`

### Step 3: 写入命令

假设设备接受 HEX 命令：

1. 选择 "HEX" 格式
2. 输入 `FF 01`
3. 点击"写入"按钮

如果成功，日志会显示：

```
✅ 写入成功 - FF 01
```

---

## 六、订阅通知（1 分钟）

### Step 1: 找到可通知特征值

带有"Notify"属性的特征值

### Step 2: 启用通知

点击"启用通知"按钮

### Step 3: 接收数据

当设备状态变化时，你会实时收到通知：

```
🔔 收到通知 - 01 00 00
```

---

## 七、常见问题

### Q1: 扫描不到设备？

**检查清单**：
- [ ] 设备是否开启蓝牙广播
- [ ] 设备是否被其他手机连接
- [ ] 手机蓝牙是否打开
- [ ] 是否开启了定位权限（Android 需要）

### Q2: 连接失败？

**可能原因**：
- 设备距离太远（靠近试试）
- 设备已被其他设备连接（断开重试）
- 设备电量不足

### Q3: 写入失败？

**检查清单**：
- [ ] 特征值是否支持写入
- [ ] 数据格式是否正确
- [ ] 是否需要先订阅通知

### Q4: 通知收不到？

**检查清单**：
- [ ] 是否启用了通知
- [ ] 设备是否主动发送通知
- [ ] 连接是否稳定

## 八、进阶技巧

### 1. 信号强度过滤

如果你想找特定信号强度的设备：

1. 点击扫描页面的"过滤"按钮
2. 设置 RSSI 范围（如 -70 ~ 0）
3. 只有符合要求的设备才会显示

### 2. 设备名称过滤

想找特定设备：

1. 点击"过滤"
2. 输入设备名称前缀
3. 只显示匹配的设备

### 3. 保存常用设备

连接过的设备会自动保存，下次可以：

1. 在首页点击"已连接设备"
2. 快速重新连接

## 九、实战案例

### 案例：控制 ESP32 LED

假设你有一个 ESP32 开发板：

**步骤 1**: 扫描并连接 ESP32

**步骤 2**: 找到 LED 服务 (UUID: 4FAFC201...)

**步骤 3**: 写入控制命令

| 效果 | HEX 命令 | UTF-8 命令 |
|------|----------|-----------|
| 开灯 | FF 01 | on |
| 关灯 | FF 00 | off |
| 快闪 | FF 02 | blink_fast |
| 慢闪 | FF 03 | blink_slow |

**步骤 4**: 订阅状态特征值

实时查看 LED 状态变化
