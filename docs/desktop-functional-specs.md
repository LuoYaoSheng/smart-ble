### 2.2 功能清单

#### 2.2.1 过滤面板

| 功能 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 信号强度过滤 | 滑块 | -100 dBm | 只显示不低于设定值的设备 |
| 快捷按钮 | 按钮 | - | All/-90/-70/-50 dBm |
| 名称前缀过滤 | 输入框 | 空 | 只显示名称以该前缀开头的设备 |
| 隐藏无名称设备 | 复选框 | false | 过滤掉没有本地名称的设备 |
| 重置按钮 | 按钮 | - | 恢复所有过滤条件到默认值 |

#### 2.2.2 扫描功能

| 功能 | 说明 |
|------|------|
| 开始/停止扫描 | 点击按钮切换状态 |
| 自动停止 | 扫描开始后 5 秒自动停止 |
| 节流处理 | 1 秒间隔更新设备列表，防止 UI 卡顿 |
| 设备去重 | 同一设备只显示最新信息 |
| 设备数量限制 | 最多显示 100 个设备（按信号强度排序） |
| 扫描状态显示 | 按钮文字和颜色反映当前状态 |

#### 2.2.3 设备列表

| 显示项 | 说明 |
|--------|------|
| 设备名称 | 本地名称，无名称显示 "Unknown" |
| 设备 ID | 显示后 12 位（完整 ID 在详情中） |
| 信号强度 | dBm 值 + 信号格数（1-4 格） |
| 连接按钮 | 右侧 "Connect" 按钮，已连接时显示 "Connected" |

#### 2.2.4 设备卡片交互

| 操作 | 触发方式 | 效果 |
|------|----------|------|
| 查看设备详情 | 点击设备卡片（非按钮区域） | 弹窗显示广播数据详情 |
| 连接设备 | 点击 "Connect" 按钮 | 连接设备，跳转到详情页 |

### 3.2 功能清单

#### 3.2.1 详情头部

| 元素 | 说明 |
|------|------|
| 返回按钮 | 点击返回设备列表页，保持连接状态 |
| 设备名称 | 显示当前连接的设备名称 |
| 连接状态 | ● 已连接（绿色）/ ○ 未连接（灰色） |

#### 3.2.2 操作按钮

| 按钮 | 显示条件 | 功能 |
|------|----------|------|
| Connect | 未连接时 | 连接当前设备 |
| Disconnect | 已连接时 | 断开连接，返回列表 |
| Clear Logs | 始终显示 | 清空操作日志 |

#### 3.2.3 服务列表

| 功能 | 说明 |
|------|------|
| 自动发现 | 连接成功后自动获取服务列表 |
| 展开/折叠 | 点击服务头部切换显示 |
| 特征值列表 | 显示该服务下的所有特征值 |
| 特征值属性 | Read/Write/Notify 标识 |

#### 3.2.4 特征值操作

| 操作 | 条件 | 说明 |
|------|------|------|
| Read | 特征值支持 Read | 读取特征值当前值，显示在日志中 |
| Write | 特征值支持 Write | 弹出输入框，写入数据 |
| Listen/Stop | 特征值支持 Notify | 开启/停止通知监听 |

#### 3.2.5 写入数据弹窗

```
┌─────────────────────────────────────┐
│  Write Characteristic         [×]   │

│  Characteristic: 0000FFE1-...       │
│  ┌───────────────────────────────┐  │
│  │ 请输入数据                     │  │
│  │ [_________________________]   │  │
│  └───────────────────────────────┘  │
│  ○ HEX  ○ UTF-8                   │
├─────────────────────────────────────┤
│  [Cancel]          [Write]         │

```

#### 3.2.6 操作日志

| 类型 | 颜色 | 格式 |
|------|------|------|
| system | 蓝色 | [HH:mm:ss] system 消息内容 |
| error | 红色 | [HH:mm:ss] error 错误信息 |
| read | 绿色 | [HH:mm:ss] read 数据内容 |
| write | 橙色 | [HH:mm:ss] write 数据内容 |
| receive | 紫色 | [HH:mm:ss] receive 接收数据 |

### 3.3 交互流程

#### 连接流程

```
1. 用户在列表页点击 [Connect]
       ↓
2. 跳转到设备详情页
       ↓
3. 显示 "连接中..." 状态
       ↓
4. 连接成功/失败处理
       ↓
5. 自动发现服务和特征值
```

#### 断开流程

```
1. 用户点击 [Disconnect]
       ↓
2. 断开 BLE 连接
       ↓
3. 清空服务列表
       ↓
4. 返回设备列表页
```

#### 返回按钮行为

| 当前状态 | 点击返回按钮效果 |
|----------|------------------|
| 已连接 | 保持连接，返回列表页 |
| 未连接 | 直接返回列表页 |

## 7. 数据结构

### 7.1 设备信息

```typescript
interface Device {
  id: string;           // 设备唯一标识
  name: string;         // 设备名称（可能为空）
  rssi: number;         // 信号强度 dBm
  connected: boolean;   // 是否已连接
  serviceUuids?: string[]; // 广告服务 UUID
  advData?: string;     // 广播数据 Hex
}
```

### 7.2 服务信息

```typescript
interface Service {
  uuid: string;         // 服务 UUID
  name: string;         // 服务名称
  characteristics: Characteristic[]; // 特征值列表
}
```

### 7.3 特征值信息

```typescript
interface Characteristic {
  uuid: string;         // 特征值 UUID
  name: string;         // 特征值名称
  properties: string[]; // 属性: read, write, notify
  value?: string;       // 当前值
}
```

### 7.4 日志条目

```typescript
interface LogEntry {
  time: string;         // HH:mm:ss
  type: 'system' | 'error' | 'read' | 'write' | 'receive';
  message: string;      // 日志内容
}
```

## 8. 错误处理

### 8.1 常见错误

| 错误 | 处理方式 |
|------|----------|
| 蓝牙未开启 | 提示用户开启蓝牙 |
| 无蓝牙权限 | 提示用户授权 |
| 连接超时 | 10秒超时，提示重试 |
| 连接失败 | 显示错误信息，保留在详情页 |
| 断开失败 | 记录日志，重置状态 |

### 8.2 用户提示

- 成功操作：绿色日志条目
- 失败操作：红色日志条目 + 错误信息
- 进行中：加载状态或状态文字
